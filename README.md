## /docs 폴더를 만들지 않는 이유

원본 문서(HWP, PDF, TXT 등)를 복사한 `/docs` 폴더를 만들지 않는다.

- **원본이 이미 있다**: 보고서·참고자료는 각자의 로컬에 있고 경로는 `CLAUDE.md`에 명시돼 있다.
- **저장소가 무거워진다**: 대용량 바이너리가 git 이력에 쌓이면 clone·pull 속도가 크게 느려진다.
- **이중관리 문제**: 원본 갱신 시 복사본도 함께 업데이트해야 해서 어느 쪽이 최신인지 판단 불가.

대안: 파일 경로는 `CLAUDE.md`에 기록하고, 세부 내용이 필요할 때 `mcp__kordoc__parse_pages`로 원본을 직접 파싱한다.

## 재사용 가능한 프롬프트 모음

### HWP / PDF → TXT 변환

다음 경로에 있는 [.hwp / .pdf] 파일을 파싱해서 .txt로 변환해줘.

**입력 경로:** [절대경로]  
**출력 경로:** [절대경로] (없으면 생성해줘)  
**대상 파일:** 파일명에 "[키워드]"가 포함된 .hwp 및 .pdf 파일  
**제외 파일:** `._`로 시작하는 macOS 메타데이터 파일은 제외  

**특이사항:**
- 파일명이 한글이라 macOS NFD 인코딩 이슈가 있을 수 있으니 `unicodedata.normalize('NFC', ...)` 처리 필요
- HWPX 포맷(ZIP 기반)이 있을 수 있으니 magic byte로 판별해서 처리 (`PK\x03\x04`면 ZIP)
