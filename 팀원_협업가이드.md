# 71기 3분임 — 팀원 협업 가이드

> 대상: 관리 담당자 / 프로그램 담당자 (코딩 경험 없는 팀원용, Windows 환경 기준)
> 작성: 인프라 담당 (팀 GitHub 관리자)

---

## 1. 이 프로젝트가 어떻게 돌아가는가

우리 팀은 **Claude Code**라는 AI 도구를 사용해서 자료조사·보고서 작성을 지원받습니다.  
각자 담당 파트 폴더 안에서만 작업하고, 완료 후 GitHub에 올리면 팀장이 통합합니다.

```
nhi71-team-report/          ← 프로젝트 전체 루트
├── CLAUDE.md               ← AI가 읽는 공통 지침 (팀 전체 규칙)
└── 자료조사/
    ├── 관리/               ← 관리 담당자 작업 공간
    │   ├── CLAUDE.md       ← AI가 읽는 관리 파트 지침
    │   └── (작업 파일들)
    ├── 프로그램/           ← 프로그램 담당자 작업 공간
    │   ├── CLAUDE.md       ← AI가 읽는 프로그램 파트 지침
    │   └── (작업 파일들)
    └── 인프라/             ← 인프라 담당자 작업 공간 (팀장)
        └── ...
```

**핵심 원칙**
- 자신의 담당 폴더(`관리/` 또는 `프로그램/`) 안에서만 파일을 만들고 수정합니다.
- 다른 파트 폴더나 루트 `CLAUDE.md`는 건드리지 않습니다.
- 작업이 끝나면 GitHub에 올리고 인프라 담당에게 알립니다.

**중요**: 이 프로젝트는 인프라 파트 작업이 이미 진행 중입니다. 처음부터 시작하는 것이 아니라 기존 구조에 합류하는 것이므로, 반드시 자신의 담당 폴더 안에서만 작업해야 합니다.

### CLAUDE.md 수정 권한 구조

프로젝트 내 CLAUDE.md 파일에는 수정 권한이 지정되어 있습니다.

| 파일 | 수정 가능한 사람 |
|------|----------------|
| 루트 `CLAUDE.md` | **팀장(인프라 담당)만** |
| `자료조사/인프라/CLAUDE.md` | **팀장(인프라 담당)만** |
| `자료조사/관리/CLAUDE.md` | 관리 담당 + 팀장 |
| `자료조사/프로그램/CLAUDE.md` | 프로그램 담당 + 팀장 |

- AI에게 루트 CLAUDE.md나 인프라 CLAUDE.md 수정을 요청하지 않습니다.
- 팀장이 공통 지침을 업데이트하면 담당 파트 CLAUDE.md에 하향 반영될 수 있습니다.
- 반대 방향(관리·프로그램 → 루트·인프라)은 불가합니다.

## 2. Claude Code 처음 설치하기 (Windows)

### 2-1. Git 설치

1. 인터넷에서 **"Git for Windows"** 검색 → 공식 홈페이지(git-scm.com)에서 설치 파일 다운로드
2. 설치 중 모든 옵션을 기본값으로 유지하고 Next → 설치 완료

설치 확인: 시작 메뉴에서 **Git Bash** 검색해서 실행. 아래 입력:
```
git --version
```
버전 번호가 뜨면 설치 성공.

### 2-2. Node.js 설치

1. 인터넷에서 **"Node.js 다운로드"** 검색 → 공식 홈페이지(nodejs.org)에서 **LTS 버전** 설치
2. 기본 옵션으로 설치 완료

설치 확인: Git Bash에서 입력:
```
node --version
npm --version
```
두 개 모두 버전 번호가 뜨면 성공.

### 2-3. Claude Code 설치

Git Bash에서 입력:
```
npm install -g @anthropic-ai/claude-code
```
완료 후 확인:
```
claude --version
```
버전 번호가 뜨면 설치 성공.

### 2-4. GitHub에서 프로젝트 받기

Git Bash에서 순서대로 실행:

```bash
# 1. 바탕화면으로 이동
cd ~/Desktop

# 2. 프로젝트 복사
git clone https://github.com/seenbeomlee/nhi71-team-report.git

# 3. 프로젝트 폴더로 이동
cd nhi71-team-report
```

> 처음 clone 할 때 GitHub 로그인 팝업이 뜰 수 있습니다. 계정이 없으면 인프라 담당자에게 문의.

## 3. 매일 작업 시작하기

### 3-1. 프로젝트 폴더에서 Claude Code 열기

**방법 A — Git Bash 사용 (권장)**

시작 메뉴 → Git Bash 실행 → 아래 입력:

```bash
cd ~/Desktop/nhi71-team-report
claude
```

**방법 B — Windows 탐색기에서 바로 열기**

바탕화면 → nhi71-team-report 폴더 → 우클릭 → "Git Bash Here" 선택 → `claude` 입력

검은 화면에 `>` 프롬프트가 뜨면 준비 완료.

### 3-2. 첫 번째로 할 말 (복사해서 붙여넣기)

**관리 담당자:**
```
루트 CLAUDE.md와 자료조사/관리/CLAUDE.md를 읽어서 컨텍스트를 파악해줘. 나는 관리 파트 담당자야. 오늘 작업할 내용은 [여기에 입력].
```

**프로그램 담당자:**
```
루트 CLAUDE.md와 자료조사/프로그램/CLAUDE.md를 읽어서 컨텍스트를 파악해줘. 나는 프로그램 파트 담당자야. 오늘 작업할 내용은 [여기에 입력].
```

이렇게 시작하면 AI가 프로젝트 맥락을 전부 파악하고 도와줍니다.

## 4. 작업 예시 — 이렇게 요청하면 됩니다

### 자료조사 요청 예시
```
관리 1 쟁점(재정 지속가능성)에 필요한 자료를 웹에서 검색해줘.
지자체별 장애인 생활체육 예산 불균형 관련 통계가 있으면 찾아줘.
```

### 파일 작성 요청 예시
```
검색한 내용을 바탕으로 자료조사/관리/관리_재정지속가능성.md 파일을 만들어줘.
문제현황 → 원인분석 → 해외사례 → 개선방안 순서로 작성해줘.
```

### 기존 파일 수정 요청 예시
```
자료조사/관리/관리_재정지속가능성.md 파일을 열어서
3번 항목에 아래 내용을 추가해줘: [내용 입력]
```

> **주의**: 파일 생성·수정은 항상 자신의 담당 폴더(`관리/` 또는 `프로그램/`) 안에서만 요청하세요.

## 5. 작업 완료 후 GitHub에 올리기

작업이 끝나면 아래 절차를 따라주세요. **순서대로** 실행합니다.

### 5-1. 작업 브랜치 확인

**각 파트에 전용 작업 브랜치가 지정되어 있습니다. 반드시 아래 브랜치에서 작업하세요.**

| 파트 | 브랜치명 | 비고 |
|------|---------|------|
| 관리 담당자 | `mgmt/policy-report` | 관리 파트 전용 |
| 프로그램 담당자 | `prog/policy-report` | 프로그램 파트 전용 |
| 인프라 담당자 (팀장) | `infra/policy-report` | 인프라 파트 전용 |

Claude Code에서 아래처럼 요청하세요:

```
mgmt/policy-report 브랜치로 전환해줘.
```

브랜치 전환 확인 후 작업을 시작하세요. `git branch` 명령으로 현재 브랜치를 항상 확인합니다.

> 하나의 파트 브랜치 안에서 세부 작업이 완료되면 커밋하면 됩니다. 추가로 세부 브랜치를 만들고 싶다면 인프라 담당에게 먼저 협의하세요.

### 5-2. 커밋 (저장) 요청

```
지금까지 작업한 내용을 커밋해줘.
```

AI가 변경된 파일을 확인하고 커밋 메시지를 제안합니다. 내용이 맞으면 승인하세요.

### 5-3. GitHub에 올리기 (Push)

```
GitHub에 푸시해줘.
```

처음 push 시 GitHub 로그인 창이 뜰 수 있습니다. 로그인하면 이후에는 자동 처리됩니다.

### 5-4. 인프라 담당자에게 알리기

카카오톡 또는 문자로 아래 내용을 전달하세요:

```
[관리] 재정지속가능성 초안 작성 완료.
브랜치명: 관리/재정지속가능성-초안
GitHub 올렸어.
```

## 6. 자주 하는 실수 — 하지 말아야 할 것

| 하지 말 것 | 이유 |
|-----------|------|
| 다른 파트 폴더 수정 (`인프라/`, `프로그램/`) | 병렬 작업 충돌 발생 |
| 루트 `CLAUDE.md` 수정 | 공통 지침 훼손 — 인프라 담당만 수정 |
| `git add .` 또는 `git add -A` 입력 | 다른 사람 작업 덮어쓰기 위험 |
| 작업 도중 GitHub 강제 push | 다른 사람 작업 삭제 위험 |
| AI에게 커밋·푸시 없이 "전부 저장해줘" | 아무것도 기록되지 않음 |

## 7. Windows 환경 자주 겪는 문제

| 상황 | 해결 방법 |
|------|----------|
| Git Bash가 없음 | Git for Windows 재설치 |
| `claude` 명령어를 찾을 수 없음 | Git Bash를 관리자 권한으로 실행 후 재시도 |
| Claude Code가 실행이 안 됨 | `claude --version` 확인 후 인프라 담당 연락 |
| AI가 엉뚱한 파일을 만듦 | AI에게 "방금 만든 파일 삭제해줘"라고 요청 |
| 실수로 다른 파트 파일을 수정한 것 같음 | **즉시 작업 중단**, 인프라 담당에게 연락 |
| 에러 메시지가 뜬 경우 | 화면 캡처해서 인프라 담당에게 전송 |
| GitHub 로그인을 계속 물어봄 | Git Bash에서 `git config --global credential.helper manager` 입력 후 재시도 |
| 한글 파일명이 깨져 보임 | Git Bash 옵션 → Text → Character set: UTF-8 설정 |

## 8. 담당 파트별 작업 범위 요약

| 쟁점 | 내용 | 파일 위치 |

| 관리 2 | 홍보·정보 체계 분절 (통합 플랫폼 구축) | `자료조사/관리/` |

| 쟁점 | 내용 | 파일 위치 |
|------|------|----------|
| 프로그램 1 | 획일화 프로그램 개선 (장애유형별 맞춤 프로그램) | `자료조사/프로그램/` |
| 프로그램 2 | 전문 강사·인력 부족 해소 방안 | `자료조사/프로그램/` |

## 9. 전체 흐름 한눈에 보기

```
각자 Claude Code로 작업
        ↓
담당 폴더 안에 파일 생성·수정
        ↓
커밋 + GitHub 푸시 + 인프라 담당에게 알림
        ↓
인프라 담당이 브랜치 병합 (merge)
        ↓
통합된 내용으로 AI가 전체 보고서 분석·작성 지원
```
